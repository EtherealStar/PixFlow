# Product

## Register

product

## Platform

web

## Users

主要受众是电商运营人员:非技术背景,日常处理商品图、查看经营数据、需要运营报告,通过对话完成工作。当前项目处于演示阶段,鉴权因此限制为单管理员,但界面不暴露、不强调这一约束——所有表面都按「运营人员的工作台」来设计。

次要受众是技术人员:Rubrics 评估模块面向他们,用于评估图片质量、文案质量和决策质量。

## Product Purpose

PixFlow 把商品素材、电商数据、视觉理解、长期记忆和可编排图片处理工具整合进同一个对话工作流。运营人员用一句自然语言指令,触发编译成 DAG 的批量图片处理链(换底、裁剪、压缩、格式转换、AI 重绘),并获得有数据依据的运营分析报告。

成功的定义:演示观众或运营用户用完之后相信「它真能替运营干活」——从上传到出图的全链路真实可用,不是玩具 demo。

## Positioning

对话即批量图片流水线:一句自然语言指令,编译成可恢复、可重试的专业批处理链。每一个界面决策都要强化这一主张。

## Brand Personality

专业、克制、高效。工具感优先:信息密度合理,色彩只为状态和主操作服务。情感目标是让运营用户感到可靠与可控——任务进度随时可见,失败可以解释,结果可以追溯。

## Anti-references

- 营销 SaaS 官网风:大 hero、渐变标题、玻璃拟态卡片、无处安放的动效。
- 中后台模板风:Ant Design Pro / Element Admin 式默认蓝白表格后台,一眼模板。
- AI 炫技风:发光边框、粒子背景、紫蓝霓虹渐变,「一看就是 AI 生成的」。

## Design Principles

1. 证明它能干活。真实的进度、真实的状态、真实的文件——演示的说服力来自全链路可用,而不是界面漂亮。任何看起来「像 demo」的占位与假数据都在破坏定位。
2. 工具消失于任务。用运营人员熟悉的词汇和标准控件(素材、产物、任务),不发明新交互,不让用户察觉系统概念。赢得熟悉感,胜过制造惊喜。
3. 状态永远可见且可解释。Activity 面板与规范状态文案是信任的载体;原始枚举值、工具名、trace ID 永远不出现在界面文本里。
4. 为运营而非管理员设计。单管理员鉴权是演示阶段的约束,不是产品身份;界面语言、默认视图、空状态都面向运营的日常重复工作。
5. 克制即专业。色彩只服务于状态与主操作,密度服务于每日重复使用。每一处装饰都必须回答「它帮用户更快确认了什么」。

## Accessibility & Inclusion

WCAG 2.1 AA。全部功能键盘可达;抽屉与对话框陷阱并恢复焦点;实时进度使用克制的 ARIA live 区域,流式 token 增量不逐片播报;色彩永不作为唯一状态信号;所有动效提供 reduced-motion 备选。
